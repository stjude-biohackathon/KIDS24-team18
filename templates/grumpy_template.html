About CAB-Grumpy

CAB-Grumpy (Center for Applied Bioinformatics - Generative Research Utility Model in Python) is a tool to run Biological Context Analysis (BCA) utilizing Large Language Model (LLM) from OpenAI St. Jude Dedicated GPT-4 Instance. The BCA assessment below was generated using Grumpy, please review it carefully and skeptically as it may contain inaccurate information resulting from so called hallucinations (read more about this phenomenon in LLM models for example in PMID 36811129).

Please note that this tool is currently in its alpha stage and is under active development. Your feedback is invaluable as we work to improve its features and functionality.

What are the differences between Precise, Balanced and Creative versions of the results?
In this evaluation, CAB-Grumpy is being run in three modes that commonly correspond with three settings that tailor how much flexibility is given to the GPT-4 model to generate the 'creative' solutions.
  • Precise Mode (temperature=0.1, top_p=0.6): prioritizes accuracy, coherence, and factual correctness; It minimizes randomness and focuses on generating concise and precise answers. Use precise mode when you need factual information, technical explanations, or when you want the model to adhere closely to the input context. In practice, for most of the BCA applications, we find the results from the precise mode the most desirable.
  • Balanced Mode (temperature=0.5, top_p=0.8): strikes a middle ground between creativity and coherence; It aims to provide well-rounded responses that are both interesting and contextually relevant.
  • Creative Mode (temperature=0.85, top_p=0.9): The purpose of using creative mode is to encourage the model to generate imaginative and novel responses. In this mode, the model introduces more randomness into its output. It explores a wider range of possibilities, leading to creative and unexpected text. Creative mode is useful when you want to analyze the data from the unique perspective or brainstorm ideas. When using the results from this mode, pay extra attention to the presence of hallucinations and make sure you review the content of the 'Sanity check' below the evaluation.

What do the temperature and top_p parameters mean?
  • Temperature:
    • The temperature parameter controls the randomness of the models output. It affects how diverse and creative the generated text is.
    • Specifically, when you set a higher temperature, the model becomes more exploratory and produces more varied responses. These responses might include unexpected or imaginative content.
    • Conversely, a lower temperature makes the output more deterministic and focused. It tends to generate more predictable and coherent text.
    • The temperature value ranges from 0 (no randomness - deterministic output) to 1 (maximum randomness - highly creative, but potentially less coherent)
  • Top-p (Nucleus) Sampling:
    • The top-p (or nucleus) sampling method restricts the models choices to the most probable tokens based on their cumulative probabilities.
    • When you set a lower top-p value, the model considers only a subset of the most likely tokens. This can lead to more focused and precise responses.
    • Higher top-p value allows the model to explore a broader range of possibilities, even if some tokens have lower probabilities.
    • The top_p value ranges from 0 (consider only the most probable token - deterministic output) to 1 (consider all tokens - more exploratory)
